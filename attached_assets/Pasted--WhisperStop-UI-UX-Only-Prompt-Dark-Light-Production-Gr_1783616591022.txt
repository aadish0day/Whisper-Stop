## SCOPE — READ THIS FIRST

Build **only the front-end shell** of WhisperStop — a community-powered WhatsApp misinformation fact-checker. This pass is about design: a **modern, polished, production-ready UI** with full **light and dark mode**, not about wiring up a backend.

Hard rules:
- **No Firebase, no Gemini, no OCR, no network calls** of any kind except loading Google Fonts.
- **No Dart / Flutter.** Plain JavaScript, React + Vite.
- **All data is mock**, in a local `mockData.js` file and React state/context.
- **Both themes are first-class.** Nothing ships "dark only, light as an afterthought" — every screen, component, and state must look intentional in both.

This should read like a real product a design team shipped, not a scaffold — considered spacing, real typography hierarchy, real motion, real empty/error/loading states, accessible by default.

---

## DESIGN CONCEPT (read before building anything)

**Subject:** people forward things on WhatsApp faster than they can verify them. The product's whole value is turning a shaky forward into something with the weight of an official verdict — so the design leans into the language of **stamps, seals, and verification marks**, not a generic dashboard aesthetic.

**Signature element — the Verification Stamp:** the large verdict badge (used on the claim detail page and the shareable fact-check card) is rendered as an actual stamp: a rotated, double-ringed circular mark with letter-spaced caps text, like a rubber stamp pressed onto a document. It's the one bold, memorable thing in the interface. Every other badge (in lists, tables, cards) stays a flat, quiet pill — the stamp treatment is reserved for the moment a verdict is "delivered," so it keeps its impact instead of being everywhere.

**Type pairing:** a confident geometric display face for headings (personality, a little technical/civic authority), a highly legible body face for everything you read, and a monospace face reserved for data — timestamps, source domains, confidence numbers, claim IDs — so numbers and metadata feel verified/precise rather than decorative.

**Color logic:** dark mode is ink-black with a saturated violet accent (the existing WhisperStop identity). Light mode isn't an inverted clone — it's a warm, slightly lavender-tinted paper white, with a deepened accent purple for text/icon contrast and the brighter dark-mode purple repurposed as the light-mode *hover* state. That's a deliberate, explainable relationship between the two themes, not two unrelated palettes.

**Restraint:** one orchestrated animation (the stamp landing), otherwise motion stays quiet — hover states, fades, nothing that scatters attention.

### Spacing, radius, elevation

- Spacing scale (px, 4-based): `--space-1: 4, --space-2: 8, --space-3: 12, --space-4: 16, --space-5: 24, --space-6: 32, --space-7: 48, --space-8: 64, --space-9: 96`.
- Radius: `--radius-sm: 8px` (buttons, inputs), `--radius-md: 12px` (small components), `--radius-lg: 16px` (cards), `--radius-full: 999px` (pills, avatars).
- Elevation: dark mode leans on `1px solid var(--color-border)` plus, only on primary CTAs, a soft accent glow (`box-shadow: 0 0 24px rgba(108,99,255,0.25)`). Light mode leans on soft shadows since borders alone read flat on white: `--shadow-sm: 0 1px 2px var(--color-shadow)`, `--shadow-md: 0 4px 16px var(--color-shadow)`, `--shadow-lg: 0 12px 32px var(--color-shadow)`.

### Motion

- `--transition-fast: 120ms ease-out` — hover/press states.
- `--transition-base: 220ms cubic-bezier(0.4,0,0.2,1)` — menus, tab switches, theme cross-fade.
- `--transition-stamp: 480ms cubic-bezier(0.34,1.56,0.64,1)` — the one signature "stamp landing" animation (bouncy overshoot).
- Respect `prefers-reduced-motion: reduce` globally: disable the stamp animation and all non-essential transitions, keep only instant state changes and simple opacity fades.

### Accessibility & production-readiness checklist

- Visible focus states everywhere via `:focus-visible` — `2px solid var(--color-accent)` with `2px` offset — never suppressed.
- Every interactive element reachable by keyboard, in a sensible tab order.
- No flash of the wrong theme on load: read the saved/system theme preference synchronously before first paint (inline script in `index.html`, or a synchronous read in `main.jsx` before `ReactDOM.createRoot(...).render(...)`).
- Real responsive breakpoints: **375px** mobile, **768px** tablet, **1024px+** desktop. Be explicit per page where columns collapse (see page descriptions below).
- Every button/input has a defined hover, active/pressed, disabled, and (where relevant) loading visual state — not just a default and a hover.

## THEME SWITCHING

- `ThemeContext.jsx`: holds `theme` (`'dark' | 'light'`) and `toggleTheme()`. On first load, default to `window.matchMedia('(prefers-color-scheme: dark)')`; after that, respect whatever the person last chose, persisted to `localStorage` under `whisperstop-theme`.
- A `ThemeToggle.jsx` component: pill-shaped switch or icon button using lucide-react's `Sun`/`Moon`, placed in the `Navbar` (both desktop and the mobile menu). Toggling cross-fades the theme's colors over `--transition-base` (skip the fade under reduced motion — just snap).

## COMPONENT SPECS

### VerdictStamp.jsx (the signature element)
Used only for the "hero" verdict moment (ClaimDetail, FactCheckCard). A circular mark, `~140px` diameter: outer solid ring in the verdict color, inner dotted/perforated ring a few px in, verdict word in the display face, all caps, letter-spaced, centered. Rotated a fixed amount derived from the claim id (e.g. `-4deg` to `-7deg`, deterministic — not randomized on every render) so it reads like a hand-stamped mark. A faint radial "ink bleed" gradient in the verdict color sits behind the text at low opacity. On mount: animates in with `--transition-stamp` — starts at `scale(1.4)`/`opacity: 0`, overshoots slightly past its resting rotation, settles. Pure CSS/SVG, no image assets. Skipped (snaps straight to final state) under `prefers-reduced-motion`.

### VerdictBadge.jsx
The quiet, everyday version used in lists/tables/cards: a flat pill, verdict color as background at ~15% opacity with the same color as text, small icon (✅ ❌ ⚠️ ❓), no rotation, no stamp rings. PENDING gets a soft pulse animation (opacity 0.6↔1, 2s loop, disabled under reduced motion). Props: `verdict`, `size` (`'sm' | 'lg'` — `'lg'` is only used inside `VerdictStamp`, not as a standalone).

### ConfidenceBar.jsx
0–100 bar, color by band: `--verdict-false` under 40, `--verdict-misleading` 40–70, `--verdict-true` above 70. Percentage label in the mono face. Fills from 0 on mount over `--transition-base`.

### ClaimCard.jsx
Truncated text (80 chars), `VerdictBadge`, `ConfidenceBar`, category tag, relative time, view count. Hover: background shifts to `--color-surface-hover` and lifts with `--shadow-sm` (light mode) or a subtle border-color brighten (dark mode), over `--transition-fast`.

### ThemeToggle.jsx
Icon button, `Sun`/`Moon` from lucide-react, swaps with a quick cross-fade. Lives in the navbar (desktop) and mobile menu.

### NotificationBell.jsx
🔔 with an unread-count dot in `--verdict-false`. Dropdown lists the last 5 mock notifications; "Mark all read" updates context state.

### EmptyState.jsx
A single-color line icon (lucide-react, `--color-text-muted`) at reduced size + a short message + optional action button. Reused across VerifyQueue, Profile tabs, AdminPanel filters — consistent rather than bespoke per page.

### Skeleton.jsx
Shimmering placeholder blocks (a soft gradient sweep, `--color-surface` → `--color-surface-hover` → `--color-surface`) matching the shape of the content it's replacing, used during every simulated loading delay.

## CLIENT-SIDE LOGIC (pure functions, zero network calls)

### `src/logic/normalizeText.js`
```javascript
export function normalizeText(text) {
  return text
    .toLowerCase()
    .replace(/[^a-z0-9\s]/g, '')
    .replace(/\s+/g, ' ')
    .trim();
}
```

### `src/logic/duplicateDetection.js`
```javascript
import { normalizeText } from './normalizeText';

function wordSimilarity(textA, textB) {
  const wordsA = new Set(textA.split(' ').filter(w => w.length > 3));
  const wordsB = new Set(textB.split(' ').filter(w => w.length > 3));
  if (wordsA.size === 0 || wordsB.size === 0) return 0;
  const intersection = [...wordsA].filter(w => wordsB.has(w)).length;
  const union = new Set([...wordsA, ...wordsB]).size;
  return intersection / union;
}

export function checkDuplicate(incomingText, existingClaims) {
  const normalized = normalizeText(incomingText);
  let highestScore = 0;
  let matchedClaimId = null;

  existingClaims.forEach(claim => {
    const claimText = normalizeText(claim.text || claim.extractedText || '');
    const score = wordSimilarity(normalized, claimText);
    if (score > highestScore) {
      highestScore = score;
      matchedClaimId = claim.id;
    }
  });

  const DUPLICATE_THRESHOLD = 0.65;
  return {
    isDuplicate: highestScore >= DUPLICATE_THRESHOLD,
    matchedClaimId: highestScore >= DUPLICATE_THRESHOLD ? matchedClaimId : null,
    score: Math.round(highestScore * 100),
  };
}
```

### `src/logic/confidenceScore.js`
Same weighted formula as a real implementation (40% verifier agreement, 30% average verifier reputation, 30% average source quality) — pure math, no Firestore anywhere near it.

### `src/logic/cardGenerator.js`
```javascript
import html2canvas from 'html2canvas';

export async function generateCard(claimId) {
  const element = document.getElementById(`fact-check-card-${claimId}`);
  if (!element) return;
  const canvas = await html2canvas(element, {
    backgroundColor: '#0D0D14',
    scale: 2,
    useCORS: true,
    width: 540,
    height: 540,
  });
  const link = document.createElement('a');
  link.download = `whisper-stop-${claimId}.png`;
  link.href = canvas.toDataURL('image/png');
  link.click();
}
```

### `src/logic/formatDate.js`
Relative time formatting ("3 minutes ago", "2 hours ago", "5 days ago") from an ISO string, no library needed.

## RULES FOR THE AGENT

1. Build and visually check **every** screen in **both** themes before considering it done — this is the actual deliverable of this pass.
2. Every data-dependent view shows a real `Skeleton` loading state with a 300–800ms simulated delay.
3. Every form has client-side validation with inline error text in `--verdict-false`.
4. Protected routes redirect to `/login`; the admin route redirects to `/` for non-admins.
5. Fully responsive at 375/768/1024px+, with layout collapses as specified per page — not just a shrunk desktop view.
6. Reference only CSS variables for color, spacing, and radius — never a hardcoded hex or px value sitting outside the token system.
7. All timestamps render as relative time in the mono face.
8. The stamp animation plays once per reveal, is the only "big" motion in the app, and is fully disabled under `prefers-reduced-motion`.
9. No flash of the wrong theme on load — resolve the theme before first paint.
10. Duplicate detection runs before a new claim is added to state; a person can't verify their own claim or double-verify one, surfaced as inline messages.
11. At 3 verdicts, the consensus calculation runs immediately and the UI updates (toast, badge change, queue removal).
12. Reputation values stay clamped 0–100 wherever adjusted.
13. `FactCheckCard` keeps `id="fact-check-card-{claimId}"` for html2canvas.
14. Nowhere in the app should there be a `fetch`, `axios`, Firebase import, or API key — mock it instead.
